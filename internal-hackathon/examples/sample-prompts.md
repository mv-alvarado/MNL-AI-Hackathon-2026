# Sample Prompts Library

This collection of tested prompts can help you get started quickly. Adapt them to your specific needs.

## Summarization Prompts

### Executive Summary from Team Updates
```
You are helping a department manager prepare an executive update.

Take these team updates and create a one-page summary with:
1. Key Accomplishments (max 3)
2. Blockers Requiring Attention (with risk level: High/Medium/Low)
3. Upcoming Milestones

For each blocker, include: what it is, why it matters, what action is needed.

Use executive-friendly language. Prioritize what leadership needs to know.

[paste updates here]
```

### Meeting Notes to Action Items
```
Convert these meeting notes into a structured action items list.

For each action item, include:
- Task description (one line)
- Owner (person or role)
- Due date
- Dependencies (if any)

Group by priority: Urgent (this week), Important (next 2 weeks), Future

[paste notes here]
```

## Analysis Prompts

### Data Pattern Identification
```
Analyze this data to identify the 3 most significant patterns or anomalies.

For each pattern:
1. Describe what you observe
2. Suggest possible explanations
3. Recommend what additional data would help validate
4. Propose next actions

Focus on actionable insights, not just description.

[paste anonymized data here]
```

### Root Cause Analysis
```
Help me analyze this problem using the "5 Whys" technique.

Problem: [describe the problem]

For each level:
- Ask "why" this is happening
- Suggest possible causes
- Identify what data would confirm or rule out each cause

Goal: Identify the root cause, not just symptoms.
```

## Communication Prompts

### Email Refinement
```
Review this email draft and suggest improvements.

Audience: [describe recipients]
Goal: [what you want to achieve]

Evaluate:
1. Clarity - is the message clear?
2. Tone - is it appropriate for the audience?
3. Structure - is information in the right order?
4. Completeness - what questions might recipients have?

Provide specific suggestions, don't rewrite completely.

[paste draft here]
```

### Difficult Feedback Preparation
```
Help me prepare to give constructive feedback about [issue - no names].

Situation: [what happened]
Impact: [why it matters]
Desired change: [what you want to see]

Frame this using SBI (Situation-Behavior-Impact) model.

Also:
- Anticipate defensive reactions
- Suggest supportive follow-up actions
- Ensure tone is direct but kind
```

## Planning Prompts

### Meeting Preparation
```
Help me prepare for a meeting with [stakeholder types] about [topic].

Context: [brief background]

Provide:
1. Suggested agenda (prioritized by importance)
2. Key questions I should ask
3. Potential concerns from each stakeholder type
4. Recommended approach for areas of disagreement

Keep actionable and meeting-focused.
```

### Project Kickoff Checklist
```
Create a project kickoff checklist for [project type].

Include:
- Pre-meeting preparation
- Key topics to cover in kickoff
- Decisions that must be made
- Information to gather from team
- Next steps and owners
- Common pitfalls to avoid

Make it actionable and comprehensive.
```

## Problem-Solving Prompts

### Solution Brainstorming
```
Generate 7-10 potential solutions for this challenge:

Problem: [describe challenge]

Constraints:
- [list any constraints: budget, time, resources, etc.]

For each solution:
- Brief description
- Pros and cons
- Rough effort estimate (Low/Medium/High)
- Expected impact (Low/Medium/High)

Include a mix of quick wins and long-term solutions.
```

### Decision Matrix
```
Create a decision matrix to evaluate these options: [list options]

Evaluation criteria:
1. [criterion 1]
2. [criterion 2]
3. [criterion 3]

For each option:
- Score on each criterion (1-5 scale)
- Key strengths
- Key weaknesses
- Overall recommendation

Include rationale for scores.
```

## Learning Prompts

### Concept Explanation
```
Explain [technical concept] to a manager with no technical background.

Use:
- Simple analogies from business/daily life
- Concrete examples
- Avoid jargon (or explain when necessary)
- Visual descriptions where helpful

Goal: Make this understandable and relevant to management work.
```

### Best Practice Research
```
Research best practices for [topic/process].

Provide:
1. Overview of common approaches
2. Pros and cons of each approach
3. When to use each approach
4. Common mistakes to avoid
5. Recommended starting point for our situation

Context about our situation: [brief description]
```

## Process Prompts

### Process Documentation
```
Create step-by-step documentation for [process name].

Format each step as:
1. Step name
   - What: Brief description
   - How: Detailed instructions
   - Why: Purpose of this step
   - Common mistakes: Gotchas to avoid

Include:
- Prerequisites section
- Tools/resources needed
- FAQ section
- When to escalate or get help
```

### Workflow Optimization
```
Review this workflow and suggest improvements:

Current process:
[describe current steps]

Pain points:
[list known issues]

Suggest:
1. Steps that could be eliminated
2. Steps that could be automated
3. Steps that are out of order
4. Missing steps that should be added
5. Quick wins (easy + high impact)

Focus on practical, implementable changes.
```

## Tips for Using These Prompts

1. **Customize**: Replace bracketed placeholders with your specific details
2. **Iterate**: Start with the basic version, then refine based on results
3. **Save your best**: When a prompt works well, save it for reuse
4. **Combine**: Mix elements from different prompts for complex tasks
5. **Context matters**: Add context about your organization, team, or situation when relevant
